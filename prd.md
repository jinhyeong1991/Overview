# [PRD] 렌탈 상품 데이터 로컬 엑셀 비교기

## 1. 프로젝트 개요
* **이름:** Local Excel Diff Checker (가칭: 렌탈료 싱크 매니저)
* **한 줄 설명:** 내 PC 브라우저 안에서 두 개의 엑셀 파일(과거/현재)을 비교해 렌탈료 변동, 신규, 단종 상품을 1초 만에 발라내는 로컬 전용 웹 툴.
* **유형:** 프론트엔드 단일 페이지 애플리케이션 (Local Web SPA)
* **난이도:** 중 (로컬 브라우저에서의 데이터 파싱 및 객체 비교 로직이 핵심)

## 2. 사용자 시나리오
* **누가:** 렌탈 상품 정보(상품명, 모델명, 렌탈기간, 렌탈료, 제품코드 등)를 주기적으로 관리하고 최신화해야 하는 실무 담당자(나).
* **언제:** 기존 CMS 데이터를 전부 갈아엎지 않고, 오직 '바뀐 부분(렌탈료 변경 등)'과 '신규 추가분'만 핀포인트로 찾아내어 신규 등록/업데이트를 해야 할 때.
* **왜 쓰는지:** 수천 개의 엑셀 데이터를 일일이 눈으로 비교하는 건 불가능하니까. 두 엑셀을 던져주기만 하면 달라진 점을 자동 분류하고 색칠해서 CMS 업로드용 결과물(C)을 에러 없이 초고속으로 만들기 위해.

## 3. 핵심 기능 목록
* **필수 기능:**
    * 로컬 엑셀 파일(.xlsx) 2개(과거 양식 A, 현재 양식 B) 브라우저 드래그 앤 드롭 업로드.
    * **고유 키(Key) 생성:** `모델명` + `렌탈기간`을 묶어 하나의 고유 ID로 인식하여 병합 및 대조.
    * **데이터 4분류 식별 로직:** [업데이트(렌탈료 다름)], [신규(B에만 있음)], [단종(A에만 있음)], [유지(동일함)].
    * 비교 결과 화면 테이블 출력 및 직관적인 색상 하이라이팅.
    * 렌더링된 결과물(C)을 다시 새로운 액셀 파일(.xlsx)로 다운로드 (단종 제품은 하단 배치).
* **선택 기능:**
    * 다크모드/라이트모드 토글 스위치 (운영체제 시스템 테마 자동 감지 포함).
    * 결과 테이블 상단에 '요약 대시보드' 제공 (예: 🟢 신규 5건, 🟠 변동 12건, ⚫ 단종 3건).

## 4. 기술 스택
* **코어:** 순수 HTML, CSS, Vanilla JS (또는 React 단일 빌드 파일) -> *복잡한 개발 환경 세팅 없이 브라우저에서 더블클릭으로 바로 실행하기 위함.*
* **엑셀 파싱/추출:** `SheetJS (xlsx)` CDN -> *로컬 브라우저 단에서 엑셀을 JSON으로 읽고 쓰기 위한 필수 1티어 라이브러리.*
* **스타일링:** `Tailwind CSS` CDN -> *빠르고 세련된 UI 구현.*

## 5. 화면 구성
* **상단 (Header):** 앱 타이틀 및 우측 상단 다크모드(🌙/☀️) 전환 토글 버튼.
* **업로드 영역 (Upload Zone):** 화면을 좌우로 반 나누어 좌측은 [과거 데이터(A)], 우측은 [현재 데이터(B)] 업로드 박스 배치. 중앙에 큼직한 [비교 분석 실행] 버튼.
* **결과 뷰어 (Result Table):** 엑셀 시트와 유사한 그리드 형태의 테이블. 식별된 상태(신규/업데이트/단종/유지)를 나타내는 상태(Status) 컬럼을 맨 좌측에 고정.
* **하단 (Footer):** 우측 하단에 고정된 [결과물 엑셀 다운로드 (C)] 플로팅 액션 버튼.

## 6. 상세 기능 명세 = 요리 레시피
1. **재료 준비 (파일 읽기):** 유저가 A파일과 B파일을 넣으면, SheetJS가 브라우저 메모리 안에서 각 엑셀의 행(Row)들을 JSON 배열(데이터 덩어리)로 싹 변환해 둔다.
2. **기준점 세우기 (Key 매핑):** JSON 데이터의 각 행을 돌면서 `모델명`과 `렌탈기간`의 텍스트를 붙여 고유한 꼬리표(예: `HD-1004_36개월`)를 만든다.
3. **교집합과 차집합 찾기 (비교 분석):**
    * B에만 있는 꼬리표 ➡️ **[신규]** 마킹.
    * A에만 있는 꼬리표 ➡️ **[단종]** 마킹 (이때 기존 A의 제품코드는 유지).
    * A, B 둘 다 있는데 `렌탈료` 값이 다름 ➡️ **[업데이트]** 마킹 (기존 A의 제품코드 유지, 바뀐 렌탈료 적용).
    * A, B 둘 다 있고 렌탈료도 같음 ➡️ **[유지]** 마킹.
4. **플레이팅 (화면 출력):** 상태별로 우선순위(신규 > 업데이트 > 유지 > 단종)를 매겨 화면 테이블에 정렬해 뿌려주고, 상태에 맞는 배경색을 입혀준다.
5. **포장하기 (다운로드):** [결과물 다운로드] 버튼을 누르면, 분류된 JSON 데이터를 다시 SheetJS로 구워내어 `.xlsx` 파일(C)로 유저 PC의 다운로드 폴더에 뱉어낸다.

## 7. 디자인 가이드
* **메인 컬러 테마:** **블루(Blue, #2563EB) & 화이트(White)**. 신뢰감을 주고 눈이 편안한 B2B SaaS 느낌의 깔끔한 톤.
* **라이트 모드 (기본):** 배경 화이트(#FFFFFF), 텍스트 짙은 회색(#1E293B), 주요 버튼 및 강조 블루(#2563EB).
* **다크 모드:** 배경 깊은 네이비/블랙(#0F172A), 텍스트 밝은 회색(#F8FAFC). 블루 컬러는 다크 배경에서 쨍하게 보이도록 밝은 블루(#3B82F6)로 조정.
* **테이블 하이라이트 규칙:**
    * 업데이트(렌탈료 다름): 특정 열(렌탈료)에 옅은 **오렌지/옐로우** 배경.
    * 단종 제품: 행 전체 텍스트에 **취소선** 적용 + 옅은 회색 배경.
    * 신규 제품: 행 전체에 옅은 **블루/민트** 배경.

## 8. 전체 개발에 대한 제약사항
* **100% 로컬 구동 (Local Only):** 보안 이슈를 원천 차단하기 위해 백엔드 서버(Node.js, DB 등)를 절대 구축하지 않는다. 인터넷이 끊긴 비행기 안에서도 `.html` 파일만 열면 작동해야 한다.
* **데이터 외부 전송 금지:** 어떤 엑셀 데이터도 외부 서버나 API로 전송(fetch, ajax)되지 않도록 콘솔 및 네트워크 요청을 철저히 막는다.
* **퍼포먼스 최적화:** 엑셀 데이터가 1만 줄 이상 넘어가도 브라우저가 멈추지 않도록, 배열의 이중 반복문(O(n^2)) 대신 '해시 맵(Hash Map)' 또는 'Set' 자료구조(O(n))를 사용하여 비교 로직을 설계해야 한다.
